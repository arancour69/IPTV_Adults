# Contributing to IPTV_Adults

First off, thank you for considering contributing to IPTV_Adults! It's people like you that make this project such a great resource.

## 📋 Code of Conduct

This project and everyone participating in it is governed by our Code of Conduct. By participating, you are expected to uphold this code.

### Our Standards

We are committed to providing a welcoming and inspiring community for all. Please read and respect our standards:

- ✅ Use welcoming and inclusive language
- ✅ Be respectful of differing opinions, viewpoints, and experiences
- ✅ Provide and accept constructive criticism gracefully
- ✅ Focus on what is best for the community
- ✅ Show empathy towards other community members

- ❌ Harassment or discrimination of any kind
- ❌ Offensive language or personal attacks
- ❌ Unwelcome sexual attention or advances
- ❌ Trolling, insulting/derogatory comments
- ❌ Publishing others' private information

## 🚀 How Can I Contribute?

### 1. Report Broken Streams 🐛

Found a stream that isn't working? Help us fix it!

**Before reporting:**
- Test the stream in at least one player
- Check if it's already reported in [Issues](../../issues)
- Verify your internet connection is stable
- Try clearing your player's cache

**How to report:**
1. Go to [Issues](../../issues)
2. Click "New Issue"
3. Select "Bug Report" template
4. Fill in all required information

**Include in your report:**
```
- Playlist file: [e.g., AdultsHD_1]
- Channel name: [exact name from playlist]
- Player used: [e.g., VLC 3.0.20]
- Operating System: [e.g., Windows 10]
- When discovered: [e.g., 2026-05-23]
- Error message (if any): [copy exact error]
- Steps you tried: [what troubleshooting you did]
```

### 2. Suggest New Streams 💡

Want to add a new stream or channel?

**Before suggesting:**
- Test the stream yourself (it must work!)
- Verify stream quality (HD or better preferred)
- Check it doesn't already exist in our playlists
- Ensure it's legal to share in your jurisdiction

**How to suggest:**
1. Go to [Issues](../../issues)
2. Click "New Issue"
3. Select "Feature Request" template
4. Provide stream details

**Include in your suggestion:**
```
- Stream URL: [full, working URL]
- Channel name: [descriptive name]
- Quality: [HD/Full HD/4K]
- Stream type: [Live/VOD]
- Why add it: [brief reason]
- Tested with: [player name and version]
- Language/Region: [if applicable]
```

**Stream Quality Standards:**
- ✅ Minimum 720p resolution
- ✅ Stable, reliable streams
- ✅ Legal content only
- ✅ Working URLs
- ✅ Consistent uptime

### 3. Verify & Test Streams ✅

Help us maintain quality by testing streams regularly.

**You can:**
- Test existing streams in different players
- Report newly broken streams
- Suggest replacement streams
- Comment on existing issues with test results

**Testing checklist:**
- [ ] Stream loads in player
- [ ] Video plays without stuttering
- [ ] Audio is synchronized with video
- [ ] Playback is stable for at least 5 minutes
- [ ] Stream quality meets standards

### 4. Improve Documentation 📚

Found typos or unclear sections?

- Fix grammar and spelling
- Clarify confusing instructions
- Add missing information
- Update outdated links
- Improve formatting

### 5. Share Your Setup Experience 🎮

Help others by sharing:
- Successful player configurations
- Performance tips for different systems
- Workarounds for common issues
- Setup guides for specific hardware

**How to share:**
1. Use [GitHub Discussions](../../discussions)
2. Create a detailed post with:
   - Your setup (player, OS, hardware)
   - What works well
   - Tips and tricks
   - Screenshots if helpful

## 🔄 Pull Request Process

While we primarily accept issues, Pull Requests are welcome for:
- Documentation improvements
- Adding new features/scripts
- Bug fixes
- Code improvements

### Steps:
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/AmazingFeature`)
3. Make your changes
4. Commit with clear messages (`git commit -m 'Add: Amazing feature'`)
5. Push to your branch (`git push origin feature/AmazingFeature`)
6. Open a Pull Request

### PR Requirements:
- ✅ Clear, descriptive title
- ✅ Detailed description of changes
- ✅ Reference related issues
- ✅ Test your changes thoroughly
- ✅ Follow the project style

## 📝 Commit Message Guidelines

Keep commits clear and organized:

```
<type>: <subject>

<body>

<footer>
```

### Types:
- `Add:` New features or streams
- `Fix:` Bug fixes
- `Update:` Updates to existing content
- `Improve:` Code quality, performance
- `Docs:` Documentation changes
- `Remove:` Removing features/content

### Examples:
```
Add: New HD stream for Channel XYZ
Fix: Broken URL in AdultsHD_1 playlist
Update: Installation guide for Kodi
Improve: README formatting and clarity
Docs: Add troubleshooting section
```

## 🏷️ Issue Labels

We use labels to organize issues:

- `bug` - Something isn't working
- `enhancement` - New feature or improvement
- `documentation` - Improvements needed to docs
- `stream-request` - Request for new stream/channel
- `stream-broken` - Broken stream reported
- `help-wanted` - Need community assistance
- `good-first-issue` - Good for newcomers
- `wontfix` - Will not be fixed

## 📋 Issue Templates

We provide templates to make reporting easier:

### Bug Report
- Title
- Description
- Steps to reproduce

- Environment details

### Feature Request

- Detailed description
- Why it's needed
- Additional context

### Stream Suggestion
- Stream URL
- Channel name
- Quality level
- Testing details

## ⏱️ Response Times

Please be patient:
- Issues typically reviewed within **7 days**
- Simple issues may be faster
- Complex issues may take longer
- Maintainers volunteer their time
- Provide complete info for faster responses

## 🚫 What We Don't Accept

- ❌ Illegal or copyrighted content
- ❌ Spam or self-promotion
- ❌ Abusive or discriminatory language
- ❌ Commercial/paid links
- ❌ Personal credential sharing
- ❌ Duplicate issues without research

## 🔐 Security Issues

Found a security vulnerability?

**Do NOT open a public issue.** Instead:
1. Go to **Security** tab in the repository
2. Click **Report a vulnerability**
3. Provide details privately
4. Allow time for a fix before public disclosure
